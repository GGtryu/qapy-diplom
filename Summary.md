# Итоги автоматизации

## Что сделано:
- **Написаны UI-тесты** для сценариев покупки тура (дебет, кредит, неверная карта).
- **Интегрирован Allure** для генерации отчётов.
- **Настроена работа с БД** в рамках UI-тестов (проверка статуса операции).
- **Настроено окружение** с помощью `docker-compose` и `application.properties`.
- **Обнаружен и учтён баг** в приложении `aqa-shop` (всегда возвращает "Успешно").

## Что не реализовано:
- **API-тесты** — не входили в финальное задание.
- **Настройка CI/CD** — не требовалась по заданию.

## Причины, по которым что-то не было реализовано:
- Ограничения по времени и фокус на UI-тестирование.
- Отсутствие необходимости в настройке CI/CD для сдачи.

## Сработавшие риски:
- Долгое разбирательство с подключением к БД и веб-приложению (`Access denied`, `net::ERR_CONNECTION_REFUSED`).
- Проблемы с ожиданием элементов UI из-за асинхронной природы страницы.
- Баг в `aqa-shop`, повлиявшйший на тестирование негативных сценариев.

## Общий итог по времени:
- **Запланировано:** 40 часов
- **Потрачено:** ~50 часов
- **Обоснование расхождения:** Время дополнительно потрачено на решение технических проблем с настройкой окружения и обнаружение/обход багов в тестируемом приложении.

## Выводы:
- Тесты написаны и протестированы.
- Отчёты информативны.
- Выявлен баг в приложении, подтверждающий эффективность тестирования.
